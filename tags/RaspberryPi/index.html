RaspberryPi 基本设置

系统安装

树莓派本质上只是一块焊好CPU的主板，要想启动树莓派还需要为其装上操作系统，为了顺利完成系统的安装，需要以下设备：

  1. 路由器
  2. SD卡
  3. 电脑

系统安装主要有两种方式，一种是直接刻录，另外一种则是利用树莓派基金会开发的工具NOOBS. 对于刻录安装这种安装方式， 就是将选择系统镜像，将其下载后刻录到SD上；而是用NOOBS则是进入引导模式，根据用户的选择安装，因此离线版则是把所有的系统都下载好，而在线版则是根据选择再下载。

因为我没有外接显示器，鼠标和键盘，所以我选择了直接刻录安装的方式。

系统镜像文件可以在树莓派官网下载，同时可以参考官网的安装参考流程。

SSH 连接

将刻录完的SD卡插入树莓派的卡槽，连接上网线，在接通电源后，树莓派便开始启动了。如果没有外接显示器，这时候根本无法获知树莓派的状态，不过可以观察一下其LED灯与网口LED灯的状态。如果系统成功启动的话，路由器上将显示树莓派接入了局域网，这时候就可以使用SSH连接了。

Hello 树莓派

家里的WD Mycloud作为NAS使用，存储着各种照片，电影，电脑备份等几乎所有的数据，同时兼备了远程下载机的功能。前几天收到WD发来的邮件，提示将Mycloud升级到WD最新推出的My Cloud OS3。看着清新的应用功能界面，回家马上就升级，结果就悲剧了。

由于之前是3.0的版本，升级到4.x的固件采用了64位，导致之前在MC上安装的软件全部失效。好吧，没有就没有了，那就重新装吧。结果发现很多软件在4.x下都不能正常安装，在nasyun论坛上搜索了一下，结果都是哀声一片。看到有很多人说可以降级到3.x，好吧，折腾就折腾吧，重新刷固件降级到3.x就好，结果更加杯具，现在回头想想，当时真的太天真了。

在降级的时候就给跪了，没有办法连接到MC，不知道哪里出现问题，然后各种搜索各种折腾，最后终于还是恢复到最新版本，好了，就这样吧。作为存储了各种资料的设备，我也不太想折腾它了，搞不好里面的数据没了，得不偿失啊~~

最近在刷知乎，看到python写爬虫，之前一直想玩玩的，看到别人的回答看起来挺简单的，决定试一试，结果在windows下写脚本，各种不顺利。想来想去家里没有一台linux设备真的不方便啊。

突然想起之前一直想买的树莓派已经出第二代了，性能提升了6倍，却还是老价格35刀。好吧，也不算太贵，加上最近工作也不忙，买一个回来玩一玩吧。然后立马淘宝，挑了一下就下单，没想到第二天就送到。
